🦇 NoMoreCapture (Lilith)
📖 项目简介
NoMoreCapture 是一款基于纯 C 语言和原生 Windows API 开发的硬核“防截图/防录屏”工具。

由于 Windows 内核的安全限制，普通程序无法直接修改其他进程的窗口属性。本项目通过底层的 DLL 注入 与 Shellcode 构建 技术，强行突破限制，修改目标窗口的显示亲和性（Display Affinity），让指定的窗口在各类截图软件、录屏工具（如 OBS、微信/QQ 截图）甚至屏幕共享中完美“隐身”或变成黑块。

本项目既是一个实用的隐私保护工具，也是一个极其优秀的 Windows 核心编程、进程注入与 Win32 界面自绘的学习范例。

✨ 核心功能
🎯 动态目标捕获 (Finder Tool)： 类似 Spy++ 的雷达拖拽功能。只需按住控制面板上的雷达图标，拖动到任意目标窗口上释放，即可自动识别窗口句柄/类名并将其加入保护策略列表。

🛡️ 双重保护模式：

白名单模式： 仅对列表内添加的窗口生效防截图。

全局模式： 关闭白名单后，强制注入并隐藏除系统核心进程外的所有可视窗口。

👁️ 可视化效果切换：

穿透模式 (完全隐身)： 在截图中，目标窗口会完全透明（基于 WDA_EXCLUDEFROMCAPTURE）。

遮蔽模式 (显示黑块)： 在截图中，目标窗口显示为纯黑方块，保护内容的同时保留窗口占位（基于 WDA_MONITOR）。

💼 一键老板键 (Boss Key)： 支持注册系统全局热键。按下热键后，不仅瞬间隐藏/恢复目标窗口，同时也会隐蔽 Lilith 软件自身的所有界面及托盘图标。

🎨 无边框自绘 UI： 摆脱传统的 Win32 灰色边框，集成 stb_image.h 解析带 Alpha 通道的 PNG 背景图，结合 UpdateLayeredWindow API 实现丝滑的无边框半透明异形窗口。

⚙️ 核心技术细节与原理图解 (Under the Hood)
本项目不仅仅是简单调用 API。防截图的核心 API NtUserSetWindowDisplayAffinity 在底层系统调用中存在严格的权限限制，本项目通过一系列底层技术实现了完美突破。

1. 痛点：为什么需要注入？(内核级的严格校验)
在 win32kfull.sys 的底层实现中，内核会严格校验调用者与目标窗口是否属于同一进程，否则抛出 ERROR_ACCESS_DENIED 拒绝访问。

如图所示：进入 Ring0 内核层后，系统会强制检查参数中的 HWND 是否属于调用方进程，跨进程的直接修改会被拦截。这就是为什么常规程序无法改变其他程序截图属性的原因。

2. 突破限制：NoMoreCapture 的执行流程
为了绕过上述内核限制，我们必须让“目标进程自己调用这个 API”。这需要极高的执行权限与内存操作：

核心执行流：构建 Shellcode -> 获取目标句柄 -> 申请内存并写入 -> 创建远程线程触发执行。利用 CreateRemoteThread 强迫目标进程以自己的身份调用防截图 API。

3. 底层支撑：API 动态寻址与系统调用链
为了确保 Shellcode 在目标进程（无论是 32 位还是 64 位）中都能正确找到 API 入口，本项目抛弃了不稳定的硬编码：

通过解析 PE 头、可选头、导出表等结构，结合自定义 Hash 算法，动态获取目标进程中 user32.dll 的 API 绝对地址，从而安全地进入 Ring0 执行 Syscall。

4. 系统级防崩校验 (Safety First)
注入其他进程是一项危险操作。本项目内置了严格的安全机制：硬编码过滤了 explorer.exe (桌面/任务栏)、dwm.exe、cmd.exe 等系统核心进程。绝不触碰桌面背景层（Progman/WorkerW），有效防止了 DWM 合成层被打穿导致系统黑屏死机的问题。
